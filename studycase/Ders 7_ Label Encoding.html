Sıralı Kategorik Veriler ve Label Encoding

Doğru Dönüşüm: 0 / 6

Algoritma Mantığı

Kategorik veriler kendi aralarında bir hiyerarşi veya sıra belirtiyorsa (Örn: Eğitim durumu, memnuniyet seviyesi), One-Hot Encoding yerine **Label Encoding** tercih edilir. Bu yöntemde her benzersiz metin etiketine, aralarındaki derece ilişkisini bozmayacak şekilde benzersiz bir tam sayı atanır.

Kodlama Yol Haritası

Yandaki editörün içine sırayla şu mantığı kurmanız bekleniyor:

  1. Gelen `seviye` parametresini kontrol edin.
  2. Eğer seviye `"dusuk"` ise, hiyerarşinin en alt basamağı olarak geriye `0` sayısını döndürün.
  3. Eğer seviye `"orta"` ise geriye `1` sayısını, `"yuksek"` ise en üst basamak olarak `2` sayısını döndürün.
  4. Bu üç tanımlı etiket dışında bir metin gelirse, belirsizliği temsil etmek adına geriye `-1` döndürün.

Kullanılacak Sözdizimi

if (seviye === "dusuk") {
    return 0;
} else if (seviye === "orta") {
    // kodunuz...
}
